5. Api receives 200 OK response code and returns and returns participant id when posting a create participant request with non-ascii characters (!@). (test_id refference: 9)

6. Api receives 200 OK response code and returns and returns participant id when posting a create participant request with firstName value > 200 characters (test_id refference: 11)

7. Api receives 200 OK response code and returns and returns participant id when posting a create participant request with space instead of value for firstName or lastName (test_id refference: 12)

8. Api receives 200 OK response code and returns and returns participant id when posting a create participant request with a presentation_title value that has more than 50 characters (test_id refference: 17)

9. Api receives 200 OK response code and returns and returns participant id when adding a fifth presenter. It returns the correct http code when adding the 6th. (test_id refference: 19)

10. Response body contains "status": 0 when deleting inexistent participant (test_id refference: 41)

11. Api receives 200 OK response code and returns and returns participant id when updating participant with empty string. (test_id refference: 52)
